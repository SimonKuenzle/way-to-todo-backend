Nach dem Generieren, erscheinen mehrere Dateien im target-Ordner. Die wichtigsten sind:
  • Todo.java
  • TodosApi.java
Die Todo.java enthält die Definition des Todo-Objekts.
Sie wird von der TodosApi.java verwendet und kann auch in eigenen Java-Klassen importiert werden.
Es sollte beim ersten Verwenden des Quellcodes darauf geachtet werden, dass der Generator gestartet wurde, damit die Dateien überhaupt existieren. Auch sollte mit dem Befehl mvn clean sichergestellt werden, dass die Dateien neu generiert werden, nachdem die api.yml angepasst wurde.
Die TodosApi.java ist ein Java-Interface und enthält die Definition der Endpunkte.
Über das default-Schlüsselwort definiert das Interface die Standard-Implementierung mit dem Rückgabewert 501, not implemented.
Das Interface wird anschließend von einer Klasse implementiert, welche wieder manuell geschrieben werden muss, dem TodosController.
Der TodosController ist eine Spring Boot Controller-Klasse, die die TodosApi implementiert.
Dies erfolgt, indem die Methoden der TodosApi im Controller implementiert werden.
Controller-Klassen bilden die Schnittstelle zwischen der API und der Business-Logik.
Sie validieren die Parameter und Payloads und rufen die entsprechenden Methoden der Business-Logik auf.
Je nach Ergebnis der Validierung und Business-Logik wird ein HTTP-Statuscode und ein Payload zurückgegeben.
